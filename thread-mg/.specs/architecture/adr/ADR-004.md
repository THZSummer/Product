# ADR-004: 内存模拟 I/O 操作

**状态**: PROPOSED

**创建日期**: 2026-03-27

**关联 Feature**: MVP 基准测试框架 (mvp-test)

---

## 背景

I/O 密集型场景是 MVP 的两个核心测试场景之一。在实现 I/O 场景时，需要决策使用何种 I/O 模拟方式：

### 方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **真实磁盘 I/O** | 贴近真实场景 | 受磁盘类型、系统负载影响大，结果不稳定 |
| **真实网络 I/O** | 贴近网络应用场景 | 受网络状况、远程服务影响，不可控 |
| **内存模拟 I/O** | 稳定可重复，无外部依赖 | 不能完全代表真实 I/O 延迟特征 |

### 真实 I/O 的问题

```
真实磁盘 I/O 测试结果波动示例（5 次迭代）：
迭代 1: 1250 tasks/sec  (磁盘空闲)
迭代 2: 890 tasks/sec   (后台进程写入)
迭代 3: 1100 tasks/sec  (正常)
迭代 4: 450 tasks/sec   (系统更新)
迭代 5: 1050 tasks/sec  (正常)

标准差：~30%，难以得出可靠结论
```

### 内存 I/O 的优势

```
内存模拟 I/O 测试结果示例（5 次迭代）：
迭代 1: 125000 tasks/sec
迭代 2: 124800 tasks/sec
迭代 3: 125200 tasks/sec
迭代 4: 124900 tasks/sec
迭代 5: 125100 tasks/sec

标准差：< 1%，结果稳定可重复
```

---

## 决策

**I/O 密集型场景使用内存流（`ByteArrayInputStream`/`ByteArrayOutputStream`）模拟 I/O 操作，不使用真实磁盘或网络 I/O。**

### 技术实现

```java
public class IoBoundScenario implements Scenario {
    private final int dataSize; // 默认 10KB
    
    @Override
    public List<Runnable> createTasks(int count) {
        List<Runnable> tasks = new ArrayList<>();
        for (int i = 0; i < count; i++) {
            tasks.add(new SimulatedIoTask(dataSize));
        }
        return tasks;
    }
    
    private static class SimulatedIoTask implements Runnable {
        private final int size;
        private final byte[] testData;
        
        public SimulatedIoTask(int size) {
            this.size = size;
            this.testData = new byte[size];
            new Random().nextBytes(this.testData);
        }
        
        @Override
        public void run() {
            try {
                // 模拟写入操作
                ByteArrayOutputStream baos = new ByteArrayOutputStream(size);
                baos.write(testData);
                baos.toByteArray(); // 触发内部缓冲
                baos.close();
                
                // 模拟读取操作
                ByteArrayInputStream bais = new ByteArrayInputStream(testData);
                byte[] buffer = new byte[size];
                bais.read(buffer);
                bais.close();
                
            } catch (IOException e) {
                throw new RuntimeException("I/O 模拟失败", e);
            }
        }
    }
}
```

### 模拟参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| **数据大小** | 10 KB | 模拟典型 I/O 数据块 |
| **操作类型** | 读 + 写 | 模拟完整 I/O 流程 |
| **缓冲方式** | 内存缓冲 | 使用 JDK 内置流 |

## 后果

### 正面影响

| 影响 | 说明 |
|------|------|
| ✅ **结果稳定** | 不受外部环境影响，标准差 < 2% |
| ✅ **执行快速** | 内存操作比磁盘快 100-1000 倍 |
| ✅ **无外部依赖** | 不需要临时文件、网络连接 |
| ✅ **跨平台一致** | 所有平台内存行为一致 |
| ✅ **易于清理** | 无临时文件残留问题 |
| ✅ **并发安全** | 每个任务独立数据，无竞争 |

### 负面影响

| 影响 | 说明 | 缓解措施 |
|------|------|----------|
| ⚠️ **不能代表真实 I/O** | 内存操作延迟远低于真实 I/O | 明确文档说明这是"模拟"I/O 场景 |
| ⚠️ **缺少系统调用开销** | 未经过 OS I/O 子系统 | Phase 2 添加真实 I/O 场景作为补充 |
| ⚠️ **无背压测试** | 内存流不模拟慢消费者 | 这是 Reactive 场景的目标，Phase 2 覆盖 |

### 性能对比参考

| 操作类型 | 内存模拟 | 真实磁盘 (SSD) | 真实网络 (本地) |
|---------|---------|---------------|----------------|
| 10KB 写入 | ~0.01 ms | ~0.5 ms | ~0.2 ms |
| 10KB 读取 | ~0.01 ms | ~0.3 ms | ~0.2 ms |
| 吞吐量 | ~100,000 ops/s | ~2,000 ops/s | ~5,000 ops/s |

**注意**：内存模拟的绝对数值与真实 I/O 不同，但**相对性能对比**（平台线程 vs 虚拟线程）仍然有效。

## 验证策略

### 结果正确性验证

```java
@Test
public void testIoScenario_correctness() {
    IoBoundScenario scenario = new IoBoundScenario(10240); // 10KB
    ExecutorService executor = Executors.newFixedThreadPool(4);
    
    ScenarioResult result = scenario.execute(executor, 100);
    
    // 验证所有任务完成
    assertThat(result.getCompletedTasks()).isEqualTo(100);
    assertThat(result.isSuccess()).isTrue();
    
    executor.shutdown();
}
```

### 性能对比有效性验证

虽然绝对数值与真实 I/O 不同，但虚拟线程相对平台线程的优势应该体现：

```
预期结果模式：
- 平台线程 I/O 场景：~50,000 tasks/sec
- 虚拟线程 I/O 场景：~100,000 tasks/sec
- 提升幅度：~100% (虚拟线程更适合 I/O 密集任务)
```

## 后续演进

### Phase 2 真实 I/O 场景

在 Phase 2 中，可以添加真实 I/O 场景作为补充：

```java
public class RealIoBoundScenario implements Scenario {
    private final Path tempDir;
    
    @Override
    public ScenarioResult execute(ExecutorService executor, int taskCount) {
        List<Runnable> tasks = new ArrayList<>();
        for (int i = 0; i < taskCount; i++) {
            tasks.add(new RealFileIoTask(tempDir.resolve("test-" + i + ".tmp")));
        }
        // ... 执行并收集结果
    }
    
    @After
    public void cleanup() {
        // 清理临时文件
        Files.walk(tempDir)
            .sorted(Comparator.reverseOrder())
            .forEach(path -> Files.deleteIfExists(path));
    }
}
```

**注意事项**：
- 需要管理临时文件生命周期
- 测试前清理磁盘缓存影响
- 结果波动较大，需要更多迭代次数
- 不适合 CI/CD 环境（磁盘 IO 不一致）
