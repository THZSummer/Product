# ADR-003: 单线程顺序执行测试

**状态**: PROPOSED

**创建日期**: 2026-03-27

**关联 Feature**: MVP 基准测试框架 (mvp-test)

---

## 背景

在基准测试执行策略上，有两种选择：

1. **并行执行**：同时运行多个场景×策略组合
2. **顺序执行**：依次运行每个场景×策略组合

### 并行执行的潜在问题

| 问题 | 说明 |
|------|------|
| **资源竞争** | 多个测试同时执行会竞争 CPU、内存资源 |
| **结果干扰** | 一个测试的 GC 可能影响另一个测试的结果 |
| **同步复杂度** | 需要额外的同步机制协调测试结果收集 |
| **调试困难** | 并发问题难以复现和定位 |

### 顺序执行的特点

| 特点 | 说明 |
|------|------|
| **资源独占** | 每个测试独占系统资源，结果更准确 |
| **实现简单** | 无需并发协调，代码简单 |
| **易于调试** | 问题容易复现和定位 |
| **执行时间较长** | 总测试时间是各测试时间之和 |

---

## 决策

**每个场景×策略组合按顺序执行，不并行运行多个测试组合。**

### 执行流程

```
┌─────────────────────────────────────────────────────────┐
│                  基准测试执行流程                        │
├─────────────────────────────────────────────────────────┤
│  1. CPU 场景 × 平台线程                                  │
│     └─> 预热 (3 次) → 测试 (5 次) → 收集结果              │
│                                                         │
│  2. CPU 场景 × 虚拟线程                                  │
│     └─> 预热 (3 次) → 测试 (5 次) → 收集结果              │
│                                                         │
│  3. I/O 场景 × 平台线程                                  │
│     └─> 预热 (3 次) → 测试 (5 次) → 收集结果              │
│                                                         │
│  4. I/O 场景 × 虚拟线程                                  │
│     └─> 预热 (3 次) → 测试 (5 次) → 收集结果              │
│                                                         │
│  5. 生成对比报告                                         │
└─────────────────────────────────────────────────────────┘
```

### 核心实现

```java
public class BenchmarkRunner {
    
    public List<BenchmarkResult> runAll(
            List<Scenario> scenarios,
            List<ThreadStrategy> strategies) {
        
        List<BenchmarkResult> allResults = new ArrayList<>();
        
        // 顺序执行每个组合
        for (Scenario scenario : scenarios) {
            for (ThreadStrategy strategy : strategies) {
                System.out.println("Running: " + scenario.getName() + 
                                   " × " + strategy.getName());
                
                BenchmarkResult result = run(scenario, strategy);
                allResults.add(result);
                
                // 每轮之间短暂休息，让系统恢复
                Thread.sleep(1000);
            }
        }
        
        return allResults;
    }
    
    private BenchmarkResult run(Scenario scenario, ThreadStrategy strategy) {
        // 预热阶段
        for (int i = 0; i < config.getWarmupIterations(); i++) {
            runIteration(scenario, strategy);
        }
        
        // 测试阶段
        List<ScenarioResult> results = new ArrayList<>();
        for (int i = 0; i < config.getBenchmarkIterations(); i++) {
            results.add(runIteration(scenario, strategy));
        }
        
        return metricCollector.aggregate(scenario, strategy, results);
    }
}
```

## 后果

### 正面影响

| 影响 | 说明 |
|------|------|
| ✅ **结果准确性高** | 每个测试独占系统资源，减少干扰 |
| ✅ **实现简单** | 无需并发协调，代码量减少 30-40% |
| ✅ **易于调试** | 问题容易复现，日志清晰 |
| ✅ **资源消耗低** | 同一时间只有一个测试运行 |
| ✅ **结果可重复** | 顺序执行更容易获得一致结果 |

### 负面影响

| 影响 | 说明 | 缓解措施 |
|------|------|----------|
| ⚠️ **总测试时间较长** | 4 个组合 × (3 预热 + 5 测试) ≈ 2-3 分钟 | MVP 可接受，Phase 2 可添加并行选项 |
| ⚠️ **系统状态漂移** | 后续测试可能受前序测试影响（如 JIT 优化） | 组合之间添加短暂休息，让系统恢复 |
| ⚠️ **无法测试并发场景** | 不能模拟真实高并发场景 | Phase 2 添加"高并发短任务"场景专门测试 |

### 测试时间估算

| 测试组合 | 预热 (3 次) | 测试 (5 次) | 合计 |
|---------|------------|------------|------|
| CPU × Platform | ~15 秒 | ~25 秒 | ~40 秒 |
| CPU × Virtual | ~15 秒 | ~25 秒 | ~40 秒 |
| I/O × Platform | ~10 秒 | ~20 秒 | ~30 秒 |
| I/O × Virtual | ~10 秒 | ~15 秒 | ~25 秒 |
| **总计** | | | **~2 分 15 秒** |

符合 spec.md NFR-002 要求（单次测试 ≤ 60 秒，这里指单个组合）。

## 后续演进

### Phase 2 并行执行扩展

当需要支持并行执行时，可按以下方式扩展：

```java
public class BenchmarkRunner {
    
    // 新增并行执行方法
    public List<BenchmarkResult> runParallel(
            List<Scenario> scenarios,
            List<ThreadStrategy> strategies,
            int parallelism) {
        
        ExecutorService runnerExecutor = Executors.newFixedThreadPool(parallelism);
        List<Future<BenchmarkResult>> futures = new ArrayList<>();
        
        for (Scenario scenario : scenarios) {
            for (ThreadStrategy strategy : strategies) {
                futures.add(runnerExecutor.submit(
                    () -> run(scenario, strategy)));
            }
        }
        
        // 收集结果
        List<BenchmarkResult> results = new ArrayList<>();
        for (Future<BenchmarkResult> future : futures) {
            results.add(future.get());
        }
        
        runnerExecutor.shutdown();
        return results;
    }
}
```

**注意事项**：
- 并行执行时需要隔离资源（如不同的临时目录）
- 需要更复杂的统计处理（排除异常值）
- 结果报告需要标注执行模式（顺序/并行）
