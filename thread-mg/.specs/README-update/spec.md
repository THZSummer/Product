## 1. 上下文理解

### 1.1 项目背景
`thread-mg` 是一个 Java 线程管理基准测试项目，旨在对比不同类型线程技术的性能差异。当前 README 文件虽然提供了基本信息，但缺乏清晰的工程结构划分和验证方案。

### 1.2 问题陈述
- 当前项目结构未明确按**应用场景 + 线程技术类型**进行组合划分
- 缺少系统性的验证方案说明
- 代码实现指南不清晰

### 1.3 改进目标
通过重构 README 文件，使项目结构更加清晰，便于开发者理解和贡献。

### 6.3 验证方案设计

#### 6.3.1 验证维度

| 维度 | 指标 | 工具 | 频率 |
|------|------|------|------|
| **性能** | 吞吐量、延迟 (P50/P95/P99) | JMH | 每次构建 |
| **正确性** | 功能正确、结果一致 | JUnit 5 | 每次提交 |
| **资源** | CPU 使用率、内存占用 | JFR | 每周 |
| **稳定性** | 错误率、崩溃率 | 自定义监控 | 持续 |

#### 6.3.2 验证流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   代码提交   │ →  │  单元测试   │ →  │ 基准测试   │ →  │  报告生成   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                        ↓                    ↓                    ↓
                   通过/失败           性能回归检测          可视化展示
```

#### 6.3.3 通过标准

| 检查项 | 标准 | 处理 |
|--------|------|------|
| 单元测试 | 100% 通过 | 失败则拒绝合并 |
| 性能回归 | 下降 ≤ 5% | 超阈值需要说明 |
| 代码覆盖 | ≥ 80% | 低于阈值需要补充 |
| 文档完整 | README 已更新 | 缺失则拒绝合并 |

## 7. 边界情况 (Edge Cases)

### EC-001: 新增场景
- **场景**: 需要添加新的测试场景
- **处理**: 在 scenarios 目录创建新子目录，更新 README 结构矩阵

### EC-002: 新增线程技术
- **场景**: Java 发布新的线程技术
- **处理**: 在 threads 目录创建新子目录，更新 README 结构矩阵

### EC-003: 不支持的组合
- **场景**: 某些场景不适合特定线程技术
- **处理**: 在矩阵中标记为「-」，在文档中说明原因

### EC-004: 性能数据异常
- **场景**: 基准测试结果异常波动
- **处理**: 触发复测机制，连续 3 次一致才接受
