## 上下文

### 问题陈述
系统需要实现基于角色的访问控制（RBAC），允许管理员创建、修改、删除角色，并为角色分配权限。

### 目标用户
- 系统管理员：管理角色和权限
- 普通用户：查看自己的角色

### 相关功能
- 用户认证系统（前置依赖）
- 权限定义（前置依赖）
- 权限检查（后续功能）

> 💡 **学习点**: 清晰的上下文帮助理解"为什么"要做这个功能，明确依赖关系。

---

## 目标与非目标

### Goals ✅
- 创建、读取、更新、删除角色
- 为角色分配/移除权限
- 为用户分配/移除角色
- 查询用户的角色和权限

### Non-Goals ❌
- 角色层级/继承 - 留待 RBAC-002
- 数据级权限 - 留待 RBAC-003
- 动态权限（ABAC） - 留待 ABAC-001

> 💡 **学习点**: 明确 Non-Goals 同样重要，防止范围蔓延。将复杂功能拆分为多个 Feature。

## 功能需求

### 角色 CRUD

| ID | 需求 | 验收标准 |
|----|------|----------|
| FR-001 | 系统应支持创建新角色 | 创建后角色存在于数据库，返回角色 ID |
| FR-002 | 角色名称必须唯一（不区分大小写） | 重名创建返回 409 冲突错误 |
| FR-003 | 角色名称长度限制 3-32 字符 | 超出范围返回 400 错误 |
| FR-004 | 系统应支持修改角色信息 | 修改后数据持久化，返回更新后的角色 |
| FR-005 | 系统应支持删除角色 | 删除后角色不可查询，关联的用户 - 角色关系清除 |
| FR-006 | 删除内置角色（如 admin）应被禁止 | 返回 403 错误 |

### 权限分配

| ID | 需求 | 验收标准 |
|----|------|----------|
| FR-007 | 系统应支持为角色分配权限 | 分配后角色拥有相应权限 |
| FR-008 | 系统应支持从角色移除权限 | 移除后角色不再拥有该权限 |
| FR-009 | 应支持批量分配权限 | 一次请求可分配多个权限 |
| FR-010 | 分配不存在的权限应失败 | 返回 400 错误，列出无效权限 |

### 用户 - 角色关联

| ID | 需求 | 验收标准 |
|----|------|----------|
| FR-011 | 系统应支持为用户分配角色 | 分配后用户拥有角色的所有权限 |
| FR-012 | 系统应支持从用户移除角色 | 移除后用户不再拥有该角色的权限 |
| FR-013 | 用户可拥有多个角色 | 权限为所有角色的并集 |
| FR-014 | 应支持查询用户的所有角色 | 返回角色列表 |

### 权限查询

| ID | 需求 | 验收标准 |
|----|------|----------|
| FR-015 | 应支持查询角色的所有权限 | 返回权限列表 |
| FR-016 | 应支持查询用户的所有权限 | 返回所有角色权限的并集（去重） |
| FR-017 | 应支持检查用户是否有特定权限 | 返回 boolean |

> 💡 **学习点**: 
> - 每个需求有唯一 ID，便于追踪
> - 验收标准必须可测试
> - 包含错误场景的处理

## 技术设计

### 数据模型

```typescript
// 角色
interface Role {
  id: string;           // UUID
  name: string;         // 唯一，3-32 字符
  description?: string; // 可选描述
  isBuiltIn: boolean;   // 是否内置角色（不可删除）
  createdAt: Date;
  updatedAt: Date;
}

// 权限
interface Permission {
  id: string;           // UUID
  code: string;         // 唯一，如 "user:create"
  name: string;         // 显示名称
  description?: string;
  module: string;       // 所属模块，如 "user", "role"
}

// 用户 - 角色关联
interface UserRole {
  userId: string;
  roleId: string;
  createdAt: Date;
  createdBy: string;    // 操作者 ID
}

// 角色 - 权限关联
interface RolePermission {
  roleId: string;
  permissionId: string;
  createdAt: Date;
  createdBy: string;
}
```

### API 接口

```
# 角色管理
POST   /api/roles              # 创建角色
GET    /api/roles              # 获取角色列表
GET    /api/roles/:id          # 获取角色详情
PUT    /api/roles/:id          # 更新角色
DELETE /api/roles/:id          # 删除角色

# 角色权限
GET    /api/roles/:id/permissions          # 获取角色权限
PUT    /api/roles/:id/permissions          # 设置角色权限（全量）
POST   /api/roles/:id/permissions          # 添加权限
DELETE /api/roles/:id/permissions/:permId  # 移除权限

# 用户角色
GET    /api/users/:userId/roles            # 获取用户角色
PUT    /api/users/:userId/roles            # 设置用户角色
POST   /api/users/:userId/roles            # 添加角色
DELETE /api/users/:userId/roles/:roleId    # 移除角色

# 权限检查
GET    /api/users/:userId/permissions      # 获取用户权限
GET    /api/users/:userId/permissions/:code # 检查特定权限
```

## 边界情况

| ID | 场景 | 处理方式 |
|----|------|----------|
| EC-001 | 创建已存在的角色名 | 返回 409 Conflict |
| EC-002 | 删除不存在的角色 | 返回 404 Not Found |
| EC-003 | 删除内置角色（admin） | 返回 403 Forbidden |
| EC-004 | 分配不存在的权限 | 返回 400，列出无效权限 |
| EC-005 | 用户被分配多个有冲突的角色 | 权限取并集，无冲突概念 |
| EC-006 | 并发修改同一角色 | 使用乐观锁（version 字段） |
| EC-007 | 数据库不可用 | 返回 503，记录错误日志 |

> 💡 **学习点**: 边界情况往往被忽略，但它们是系统健壮性的关键。
