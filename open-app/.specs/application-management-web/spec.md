## 1. 概述

### 1.1 背景

**open-app** 是企业通讯能力开放平台，将 XXX 通讯系统的核心能力（IM、Meeting、CloudBox 等）开放给业务应用和个人应用。

**应用管理 Web 端** 是应用管理系统的用户界面，为应用开发者提供可视化的应用管理控制台。

### 1.2 问题陈述

应用管理 Web 端主要解决：
- **可视化操作**：为后端 API 提供友好的用户界面
- **应用管理效率**：通过列表、搜索、筛选等功能提高管理效率
- **状态可视化**：直观展示应用状态和生命周期
- **权限可视化**：清晰展示用户权限和可操作范围

### 1.3 目标用户

| 用户类型 | 说明 | 使用场景 |
|----------|------|----------|
| **应用开发者** | 创建和管理自己的应用 | 创建应用、编辑信息、查看列表、删除应用 |

### 1.4 技术栈

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| **框架** | React | 18.x | UI 框架 |
| **语言** | TypeScript | 5.x | 类型安全 |
| **构建工具** | Vite | 5.x | 快速构建 |
| **UI 组件库** | Ant Design | 5.x | 企业级 UI |
| **状态管理** | Zustand | 4.x | 轻量状态管理 |
| **路由** | React Router | 6.x | 客户端路由 |
| **HTTP 客户端** | Axios | 1.x | API 请求 |
| **样式** | Less | 4.x | CSS 预处理器 |

**模块性质**: 本模块为 **纯前端 Web 模块**，不包含任何后端逻辑。后端服务以 `-api` 或独立后端服务形式存在（如 `application-management` 后端模块）。

## 4. 功能需求

### 4.1 应用列表页 (FR-001)

**描述**: 展示用户可访问的应用列表

**需求**:
- FR-001.1: 系统必须展示应用列表，包含字段：应用名称、应用类型、状态、创建时间、更新时间
- FR-001.2: 列表必须支持分页，默认每页 20 条
- FR-001.3: 系统必须支持按应用名称模糊搜索
- FR-001.4: 系统必须支持按状态筛选（全部/草稿/已启用/已禁用/已删除）
- FR-001.5: 开发者只能看到自己的应用，管理员可以看到所有应用
- FR-001.6: 列表必须支持按创建时间排序（最新/最旧）
- FR-001.7: 点击应用名称可跳转到应用详情页

## 7. 数据流设计

### 7.1 状态管理结构

```typescript
// stores/applicationStore.ts
interface ApplicationState {
  // 列表数据
  applications: Application[];
  total: number;
  page: number;
  pageSize: number;
  
  // 筛选条件
  filters: {
    keyword?: string;
    status?: AppStatus;
    type?: AppType;
  };
  
  // 当前选中的应用
  currentApplication?: Application;
  
  // UI 状态
  loading: boolean;
  error?: string;
  
  // 操作
  fetchApplications: (params: ListParams) => Promise<void>;
  createApplication: (data: CreateAppData) => Promise<Application>;
  updateApplication: (id: string, data: UpdateAppData) => Promise<Application>;
  deleteApplication: (id: string) => Promise<void>;
  setCurrentApplication: (app?: Application) => void;
  setFilters: (filters: Partial<ApplicationState['filters']>) => void;
}
```

### 7.2 API 调用层

```typescript
// services/applicationApi.ts
const applicationApi = {
  // 获取应用列表
  getList: (params: ListParams) => 
    axios.get('/api/v1/applications', { params }),
  
  // 获取应用详情
  getDetail: (id: string) => 
    axios.get(`/api/v1/applications/${id}`),
  
  // 创建应用
  create: (data: CreateAppData) => 
    axios.post('/api/v1/applications', data),
  
  // 更新应用
  update: (id: string, data: UpdateAppData) => 
    axios.put(`/api/v1/applications/${id}`, data),
  
  // 删除应用
  delete: (id: string) => 
    axios.delete(`/api/v1/applications/${id}`),
};
```

## 9. 边界情况

### 9.1 错误处理 (EC-001)

| 场景 | 前端处理 |
|------|----------|
| API 返回 404 | 展示「应用不存在」提示，返回列表页 |
| API 返回 403 | 展示「无权限访问」提示 |
| API 返回 409 | 展示「应用名称已存在」提示 |
| API 返回 500 | 展示「系统繁忙，请稍后重试」提示 |
| 网络错误 | 展示「网络连接失败，请检查网络」提示 |
| Token 过期 | 自动跳转登录页 |

### 9.2 极端情况 (EC-002)

| 场景 | 前端处理 |
|------|----------|
| 列表数据为空 | 展示空状态组件，引导用户创建应用 |
| 列表数据过多 | 使用虚拟滚动或分页 |
| 图片加载失败 | 展示默认图标 |
| 表单数据过长 | 截断展示，详情可查看完整内容 |
| 连续快速点击 | 防抖处理，防止重复提交 |

### 9.3 并发场景 (EC-003)

| 场景 | 前端处理 |
|------|----------|
| 多 Tab 同时操作 | 操作完成后刷新列表 |
| 后台切换导致 Token 过期 | 捕获错误，跳转登录 |
| 列表加载中切换筛选条件 | 取消上一个请求，发起新请求 |
