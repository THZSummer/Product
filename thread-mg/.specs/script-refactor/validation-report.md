## 🔄 前后对比

### 代码风格统一性

| 方面 | 重构前 | 重构后 |
|------|--------|--------|
| 颜色定义 | 各脚本独立定义 | 统一使用 lib.sh |
| 日志函数 | 各脚本独立实现 | 统一使用 lib.sh |
| 参数格式 | 仅短参数 | 短参数 + 长参数 |
| 代码重复 | 高（~100 行重复） | 低（公共库） |
| 可维护性 | 低 | 高 |

### 脚本结构

**重构前**:
```bash
# 每个脚本都有重复代码
COLOR_RED='\033[0;31m'
COLOR_GREEN='\033[0;32m'
# ...
log_info() { echo -e "${COLOR_BLUE}[INFO] $1"; }
```

**重构后**:
```bash
# 统一导入公共库
SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"
source "$SCRIPT_DIR/lib.sh"

# 直接使用
log_info "消息"
```
