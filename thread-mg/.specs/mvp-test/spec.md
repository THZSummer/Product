## 1. 概述

### 1.1 背景

本项目旨在系统性地研究和对比 Java 平台上不同类型线程技术在相同场景下的性能表现。MVP 阶段目标是快速构建最小可行产品，在 1-2 周内打通"场景定义 → 线程策略 → 性能测试 → 结果报告"核心流程。

### 1.2 解决的问题

1. 缺乏统一的基准测试框架来对比不同线程技术的性能
2. 开发者难以根据具体业务场景选择合适的并发方案
3. 需要验证 Java 21 虚拟线程相对于传统平台线程的性能优势

### 1.3 目标用户

- Java 后端开发工程师
- 系统架构师
- 性能优化工程师

### 1.4 相关功能

- 与 SDD 工作流集成，遵循规范驱动开发流程
- 为后续扩展更多场景和线程策略提供框架基础

## 6. 技术设计

### 6.1 架构概览

```
┌─────────────────────────────────────────────────────────┐
│                    MVP 测试工具架构                      │
├─────────────────────────────────────────────────────────┤
│  入口层                                                  │
│  └── Main / BenchmarkApp                                │
├─────────────────────────────────────────────────────────┤
│  场景层 (Scenario)                                       │
│  ├── CpuBoundScenario (CPU 密集型：矩阵计算/素数筛选)    │
│  └── IoBoundScenario (I/O 密集型：文件读写/网络模拟)     │
├─────────────────────────────────────────────────────────┤
│  策略层 (ThreadStrategy)                                 │
│  ├── PlatformThreadStrategy (平台线程)                  │
│  ├── VirtualThreadStrategy (虚拟线程)                   │
│  └── ThreadPoolStrategy (线程池 - 可选)                 │
├─────────────────────────────────────────────────────────┤
│  框架层                                                  │
│  ├── BenchmarkRunner (运行器)                           │
│  ├── MetricCollector (指标收集：吞吐量/延迟)             │
│  ├── BenchmarkConfig (配置管理)                         │
│  └── BenchmarkResult (结果对象)                         │
├─────────────────────────────────────────────────────────┤
│  报告层                                                  │
│  ├── ConsoleReporter (控制台报告)                        │
│  └── MarkdownReporter (Markdown 报告)                    │
└─────────────────────────────────────────────────────────┘
```

### 6.2 核心接口设计

#### Scenario 接口

```java
public interface Scenario {
    String getName();
    String getDescription();
    ScenarioResult execute(ExecutorService executor, int taskCount);
    List<Runnable> createTasks(int count);
}
```

#### ThreadStrategy 接口

```java
public interface ThreadStrategy {
    String getName();
    ExecutorService createExecutor();
    void shutdown();
    int getDefaultThreadCount();
}
```

#### BenchmarkResult 数据结构

```java
public class BenchmarkResult {
    private String scenarioName;
    private String strategyName;
    private long totalTasks;
    private long durationMs;
    private double throughput;      // tasks/sec
    private double avgLatencyMs;    // 平均延迟
    private double p50LatencyMs;    // P50 延迟
    private double p95LatencyMs;    // P95 延迟
    private double p99LatencyMs;    // P99 延迟
    private Instant timestamp;
}
```

### 6.3 数据模型

无持久化存储需求，所有数据在内存中处理，报告生成后保存到文件系统。

## 7. 边界情况 (Edge Cases)

| 编号 | 边界情况 | 处理策略 |
|------|----------|----------|
| EC-001 | 任务执行超时 | 设置超时阈值，超时后终止并记录为失败 |
| EC-002 | 线程创建失败（资源不足） | 捕获异常，记录错误信息，优雅降级 |
| EC-003 | Java 版本低于 21 但使用虚拟线程 | 启动时检测版本，给出友好提示 |
| EC-004 | 测试过程中 JVM 被中断 | 使用 ShutdownHook 清理资源 |
| EC-005 | 报告写入失败（目录不存在/权限不足） | 创建目录，记录日志，不阻断测试 |
| EC-006 | 任务数量为 0 或负数 | 参数校验，抛出 IllegalArgumentException |
| EC-007 | 迭代次数配置过大 | 设置上限警告，建议合理值 |
| EC-008 | 多线程并发修改指标数据 | 使用线程安全的数据结构（ConcurrentHashMap） |

## 10. 范围边界 (Scope Boundaries)

### 10.1 MVP 包含内容

✅ **场景**
- CPU 密集型场景（矩阵计算或素数筛选）
- I/O 密集型场景（内存模拟文件读写）

✅ **线程策略**
- 平台线程策略（`Executors.newFixedThreadPool`）
- 虚拟线程策略（`Executors.newVirtualThreadPerTaskExecutor`）
- 线程池策略（可选，可配置大小）

✅ **框架功能**
- 基准运行器（BenchmarkRunner）
- 指标收集器（MetricCollector）
- 配置管理（BenchmarkConfig）
- 结果对象（BenchmarkResult）

✅ **报告功能**
- 控制台报告（ConsoleReporter）
- Markdown 报告（MarkdownReporter）

✅ **质量保障**
- 单元测试（JUnit 5）
- 代码规范检查（CheckStyle）
- 覆盖率检查（JaCoCo ≥ 80%）

### 10.2 MVP 不包含内容

❌ **场景**
- 高并发短任务场景
- 混合负载场景
- 递归分治场景
- 异步流水线场景

❌ **线程策略**
- Fork/Join 策略
- CompletableFuture 策略
- Reactive Streams 策略

❌ **高级功能**
- JMH 微基准测试集成
- HTML 报告生成
- 可视化图表
- 性能回归检测
- CI/CD 集成
- 历史数据对比

❌ **其他**
- GUI 界面
- 数据库持久化
- 分布式测试
- 远程监控
