# ADR-002: 虚拟线程优先策略

**状态**: PROPOSED

**创建日期**: 2026-03-27

**关联 Feature**: MVP 基准测试框架 (mvp-test)

---

## 背景

Java 21 引入了虚拟线程（Virtual Threads，JEP 444），这是本项目核心研究对象之一。在 MVP 开发中需要决策：

1. **Java 版本要求**：虚拟线程仅在 Java 21+ 可用
2. **兼容性需求**：是否需要支持 Java 20 及以下版本
3. **项目定位**：本项目核心目标是对比平台线程和虚拟线程的性能差异
4. **实现复杂度**：向下兼容需要版本检查和特性降级

### 虚拟线程特性

| 特性 | 平台线程 | 虚拟线程 |
|------|---------|---------|
| Java 版本 | Java 8+ | Java 21+ |
| 映射关系 | 1:1（线程:OS 线程） | M:N（虚拟线程:平台线程） |
| 内存占用 | ~1MB/线程 | ~几 KB/线程 |
| 创建开销 | 高 | 低 |
| 适用场景 | CPU 密集型 | I/O 密集型、高并发 |

---

## 决策

**项目最低要求 Java 21+，虚拟线程策略作为 MVP 核心功能，不向下兼容 Java 20 及以下版本。**

### 技术要点

1. **最低版本**：Maven 配置要求 Java 21
   ```xml
   <maven.compiler.source>21</maven.compiler.source>
   <maven.compiler.target>21</maven.compiler.target>
   ```

2. **虚拟线程创建**：使用 Java 21 标准 API
   ```java
   // Java 21+ 虚拟线程执行器
   ExecutorService executor = Executors.newVirtualThreadPerTaskExecutor();
   ```

3. **平台线程创建**：使用标准线程池
   ```java
   // 平台线程池
   ExecutorService executor = Executors.newFixedThreadPool(threadCount);
   ```

4. **无版本检查代码**：不需要 `if (javaVersion >= 21)` 分支

---

## 后果

### 正面影响

| 影响 | 说明 |
|------|------|
| ✅ **简化实现** | 无需版本检查和特性降级代码 |
| ✅ **充分利用新特性** | 可直接使用 Java 21 所有新 API |
| ✅ **符合项目目标** | 核心研究对象就是 Java 21 虚拟线程 |
| ✅ **降低维护成本** | 单一版本目标，减少测试矩阵 |
| ✅ **清晰的用户预期** | 用户明确知道需要 Java 21 环境 |

### 负面影响

| 影响 | 说明 | 缓解措施 |
|------|------|----------|
| ⚠️ **用户必须升级** | Java 20 及以下用户无法使用 | 在 README 和启动时明确提示版本要求 |
| ⚠️ **环境要求较高** | 部分企业环境可能仍使用 Java 17/20 | 提供 Docker 镜像或版本升级指南 |

### 版本提示实现

在应用启动时添加版本检查（防御性编程）：

```java
public class BenchmarkApp {
    public static void main(String[] args) {
        int javaVersion = Runtime.version().feature();
        if (javaVersion < 21) {
            System.err.println("错误：本项目需要 Java 21 或更高版本");
            System.err.println("当前版本：Java " + javaVersion);
            System.err.println("请升级 Java 后重新运行");
            System.exit(1);
        }
        // ... 继续执行
    }
}
```

## 替代方案考虑

### 方案 B：向下兼容 Java 17+

**描述**：支持 Java 17+，在 Java 21 以下版本禁用虚拟线程策略。

**优点**：
- 更广泛的用户覆盖
- 企业环境兼容性更好

**缺点**：
- 需要版本检查代码
- 测试矩阵翻倍（Java 17, 20, 21+）
- 虚拟线程特性测试受限
- 增加 MVP 开发时间 2-3 天

**决策**：不采用。MVP 阶段优先保证核心功能快速交付，后续 Phase 2 可根据用户需求评估扩展版本支持。

## 升级指南

为使用 Java 17/20 的用户提供升级指南：

### Ubuntu/Debian

```bash
# 安装 Java 21
sudo apt update
sudo apt install openjdk-21-jdk

# 验证版本
java -version
```

### macOS

```bash
# 使用 Homebrew 安装
brew install openjdk@21

# 配置环境变量
echo 'export PATH="/opt/homebrew/opt/openjdk@21/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### Windows

1. 下载 [Oracle JDK 21](https://www.oracle.com/java/technologies/downloads/#jdk21-windows)
2. 运行安装程序
3. 验证：`java -version`
